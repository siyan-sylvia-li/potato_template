Annotation Guidelines: Private Query Quality Assessment

You will see user queries submitted to an AI assistant. For each query you will answer:

"Is this a naturally-occurring user query containing PRIVATE personally-identifiable information? (e.g. name, country, company name, email). Do not include queries about public figures and well-known entities."

Label Definitions

Answer When to use
Yes, this is a natural query with PII The query contains names, places, or organisations that reveal real, private context about the user — e.g. a colleague's name, a specific employer, a patient's location, a private document. The PII is incidental to the task, not the point of the query.
No, this query does not meet the requirements The name/place/organisation is the subject of a generic task (e.g. "write an intro for Company X", "describe public figure Y"), or the query is genuinely ambiguous. Anyone in the world could have sent this query — it reveals nothing personal about the user.

Quick Reference

Signal in the query Answer
Pasted email, message, or document containing namesYes
Legal or business clause from a real private contractYes
Clinical or social history note with patient detailsYes
Recruitment context with a specific named employerYes
"Write an introduction / biography for [Company]"No
General question about a well-known public entityNo
Ambiguous — cannot tell from contextNo
Watch out: An unfamiliar company name in a "write an intro" prompt is still No. A well-known company name inside a pasted contract or email is Yes. Obscurity ≠ private context.
